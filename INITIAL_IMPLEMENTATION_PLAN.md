# LifeOS - Initial Implementation Plan

This document outlines the step-by-step implementation plan for LifeOS, an autonomous data-processing loop that leverages Coral for data extraction and the Gemini API for intelligence.

## Architecture Overview

*   **Backend:** Node.js (Express or Fastify) or Python (FastAPI). Given the JavaScript ecosystem's strength with asynchronous operations and child processes, we will proceed with **Node.js**.
*   **Frontend:** React (Vite) or Next.js with a premium, dynamic UI.
*   **Data Layer:** Coral CLI (locally installed, mapped to external APIs via SQL).
*   **Intelligence Layer:** Gemini API.

---

## Phase 1: Project Initialization & Infrastructure

**Goal:** Set up the basic project structure and core utility functions for interacting with Coral and Gemini.

### Step 1.1: Backend Setup
1.  Initialize a new Node.js project in `c:\dev\LifeOS\backend`.
2.  Install required dependencies: `express`, `cors`, `dotenv`, `@google/genai` (for Gemini API).
3.  Create a basic Express server configuration.

### Step 1.2: Frontend Setup
1.  Initialize a new React project (using Vite) in `c:\dev\LifeOS\frontend`.
2.  Set up CSS framework (Vanilla CSS or Tailwind CSS as per preference, focusing on modern aesthetics, glassmorphism, and dynamic animations).
3.  Establish basic routing and a premium dashboard layout.

### Step 1.3: Coral CLI Integration Module
1.  Create a utility class/module in the backend to spawn child processes (`child_process.exec` or `spawn`).
2.  Implement a function `executeCoralQuery(sqlQuery)` that:
    *   Spawns `coral sql "{sqlQuery}"`.
    *   Captures the standard output.
    *   Parses the resulting JSON string into JavaScript objects.
    *   Handles errors and stderr output robustly.

### Step 1.4: Gemini API Integration Module
1.  Create a utility class/module for interacting with the Gemini API.
2.  Implement a function `analyzeDataWithGemini(prompt, data)` that constructs the final prompt and returns structured JSON insights.

## Phase 2: The "Action-Bias" Tracker (Proof of Concept)

**Goal:** Implement the first agent to prove the end-to-end flow: Coral -> Backend -> Gemini -> Frontend.

### Step 2.1: GitHub Data Fetching (Coral)
1.  Implement backend service functions using the verified SQL queries:
    *   `fetchUserProfile()`: `SELECT login, id, type, name, company FROM github.user LIMIT 1`
    *   `verifyRepository()`: `SELECT name, full_name, private, created_at FROM github.repos_get WHERE owner = 'hmbitcyber' AND repo = 'lifeos-dummy-data'`
    *   `fetchRecentCommits()`: `SELECT sha, author__login, authored_date, message FROM github.commits WHERE owner = 'hmbitcyber' AND repo = 'lifeos-dummy-data' LIMIT 5`

### Step 2.2: YouTube/Consumption Data Fetching
1.  *Mock implementation initially:* Create dummy data representing YouTube watch history (e.g., hours spent on React tutorials).
2.  *Future:* Map YouTube API in Coral and write SQL queries to fetch watch history.

### Step 2.3: Gemini Analysis Logic
1.  Design a prompt template for Gemini that takes the YouTube consumption data and GitHub commit data.
2.  The prompt should ask Gemini to calculate an "Action-Bias Score" and determine if an intervention (Calendar block) is necessary.

### Step 2.4: Frontend Visualization
1.  Create a dashboard widget for the "Action-Bias Tracker".
2.  Display the consumption vs. execution ratio.
3.  Show alerts or recommended interventions generated by Gemini.

## Phase 3: The "Money Sentinel"

**Goal:** Implement financial leak detection.

### Step 3.1: Gmail Data Fetching
1.  Map the Gmail API in Coral.
2.  Write Coral SQL queries to search `gmail.messages` for keywords like "free trial", "receipt", "auto-renewal".

### Step 3.2: Budget Data Fetching
1.  Map Google Sheets API in Coral.
2.  Write Coral SQL queries to fetch current budget data from `google_sheets.rows`.

### Step 3.3: Gemini Processing & Action
1.  Pass email contents to Gemini to extract trial end dates and subscription costs.
2.  Compare costs against the budget.
3.  If action is required, map the Google Calendar API in Coral (or use standard API calls) to create a high-priority cancellation event.

## Phase 4: "Time-Tetris"

**Goal:** Implement dynamic schedule optimization.

### Step 4.1: Schedule Monitoring
1.  Continuously query `google_calendar.events` via Coral for today's schedule.
2.  Query `google_sheets.rows` for baseline non-negotiable habits (e.g., gym sessions).

### Step 4.2: Conflict Resolution
1.  Detect changes or delays in the calendar.
2.  Pass the updated schedule and requirements to Gemini.
3.  Gemini identifies "whitespace" and returns an optimized schedule.
4.  Update Google Calendar with the new time slots.
